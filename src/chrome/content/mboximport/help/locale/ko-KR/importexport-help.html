ImportExportTools NG 개요 및 메뉴

ImportExportTools NG는 Paolo "Kaosmos"의 원작을 기반으로 한 후속 확장입니다. 확장은 다음을 포함하여 다양한 파일 형식을 지원하는 여러 가져오기 및 내보내기 기능을 제공합니다. 참고: 일부 형식은 모든 기능에서 지원되지 않습니다.
- m박스
-EML
- 메일 디렉토리
- HTML
- 일반 텍스트
- CSV(쉼표로 구분된 값)
- PDF

대부분의 기능은 메시지를 중심으로 이루어지지만 Thunderbird 프로필 가져오기 및 내보내기도 지원됩니다. 또한 내보내기에는 HTML 또는 CSV 색인 파일도 포함될 수 있습니다. Thunderbird 프로필 백업은 또 다른 기능입니다.

IETng 기능에 대한 액세스는 확장 프로그램에 의해 설치된 여러 가지 메뉴를 통해 수행됩니다. 기능은 상황에 따라 다릅니다. 다음에 대한 메뉴가 추가됩니다.
- 폴더 창 컨텍스트(폴더 작업)
- 메시지 목록 컨텍스트(선택된 메시지)
- 도구 메뉴(프로필 및 백업 작업)

"도구 메뉴"에서 액세스할 수 있는 옵션은 "햄버거" 메뉴에는 없지만 향후 버전에 포함될 예정입니다. 이러한 기능은 새로운 도구 모음 버튼에서도 사용할 수 있습니다. 각 메뉴와 버튼에는 옵션 대화 상자에 액세스할 수 있는 도움말 및 옵션 메뉴 항목도 있습니다.

사용자 정의 날짜 형식

사용자 정의 날짜 옵션을 사용하면 색인의 두 파일에 사용되는 날짜 형식을 완벽하게 제어할 수 있습니다. 파일 이름 드롭다운에서 기본 형식 "날짜"와 새로운 사용자 정의 형식 "날짜(사용자 정의)"를 모두 사용할 수 있습니다. 사용자 정의 날짜는 확장된 사용자 정의 파일 이름에 사용되는 ${date_custom} 토큰과 함께 사용할 수도 있습니다. 원래(기본) 날짜 형식은 다음과 같습니다.
YYYYMMDDHHmm : 202011060808

사용자 정의 날짜 형식은 기능과 유사한 strftime() Unix와 일치하는 % 시퀀스를 활용합니다. 날짜와 시간 형식을 지정하는 함수인 strftime() 의 JavaScript 포트를 기반으로 합니다.
TH Doan의 strftime() 포트(https://thdoan.github.io/strftime/)

변환 순서 설명

순서 설명
%a 요일의 약칭.
%A 요일의 전체 이름입니다.
%b 약식 월 이름입니다.
%B 전체 월 이름입니다.
%c 현재 로케일에 대한 기본 날짜 및 시간(UTC) 표현입니다.
%C 2자리 정수로 된 세기 수(연도/100)입니다.
%d 10진수로 표시된 날짜(범위 01~31)입니다.
%e 10진수로 표시된 날짜(범위 1~31)입니다.
%F ISO 8601 날짜 형식( %Y-%m-%d 와 동일)
%G ISO 8601 십진수로 세기를 포함한 주 기반 연도입니다. 4자리 연도는 ISO 주 번호에 해당합니다( %V 참조). ISO 주 번호가 이전 또는 다음 연도에 속하는 경우 해당 연도가 대신 사용된다는 점을 제외하면 %Y 와 형식 및 값이 동일합니다.
%g %G 비슷하지만 세기가 없습니다. 즉, 2자리 연도(00-99)가 있습니다.
%H 24시간제(범위 00~23)를 사용하여 십진수로 표시되는 시간입니다. %k 도 참조하세요.
%I 12시간제(범위 01~12)를 사용하여 십진수로 시간을 표시합니다. %l 도 참조하세요.
%j 10진수로 표시된 연도의 일(범위 001~366)입니다.
%k 24시간제(범위 0~23)를 사용하여 십진수로 시간을 표시합니다. %H 도 참조하세요.
%l 12시간제를 사용하여 10진수로 시간을 표시합니다(범위 1~12). %I 도 참조하세요.
%m 월을 십진수로 표시합니다(범위 01~12).
%n 월을 십진수로 표시합니다(범위 1~12).
%M 10진수로 표현된 분(범위 00~59)
%p 주어진 시간 값에 따라 "AM" 또는 "PM"입니다. 정오는 "PM"으로 처리되고 자정은 "AM"으로 처리됩니다.
%P %p 와 비슷하지만 소문자("am" 또는 "pm")입니다.
%s Epoch 이후의 초 수, 1970-01-01 00:00:00 +0000(UTC).
%S 10진수로 표현된 초(범위 00~59)
%u 요일을 십진수(1~7 범위)로 표시하고 월요일은 1입니다. %w 도 참조하세요.
%V ISO 8601 현재 연도의 주 번호(범위 01~53). 여기서 1주는 새해에 최소 4일이 있는 첫 번째 주(즉, 첫 번째 목요일)입니다.
%w 요일을 십진수(범위 0~6)로 표시하고 일요일은 0입니다. %u 도 참조하세요.
%x 시간 없이 현재 로케일에 대한 기본 날짜 표현입니다.
%X 날짜 없이 현재 로케일에 대한 기본 시간 표현입니다.
%y 세기가 없는 십진수로 표현된 연도(범위 00~99).
%Y 세기를 포함한 십진수로 표현된 연도입니다.
%z +hhmm 또는 -hhmm 숫자 시간대(즉, UTC로부터의 시간 및 분 오프셋)입니다.
%Z 시간대 이름 또는 약어입니다.

호환성 참고사항:

  • %c - 형식화된 문자열이 약간 다릅니다.
  • %D - 구현되지 않음( %m/%d/%y 또는 %d/%m/%y 사용)
  • %e - 공백이 추가되지 않았습니다.
  • %E - 구현되지 않음
  • %h - 구현되지 않음( %b 사용)
  • %k - 공백이 추가되지 않았습니다.
  • %n - %m 과 비슷하지만 앞에 0이 없습니다(개행에는 \n 사용).
  • %O - 구현되지 않음
  • %r - 구현되지 않음( %I:%M:%S %p 사용)
    • ${subject} 메시지 제목
  • %R - 구현되지 않음( %H:%M 사용)
  • %t - 구현되지 않음( \t 사용)
  • %T - 구현되지 않음( %H:%M:%S 사용)
  • %U - 구현되지 않음
  • %W - 구현되지 않음
  • %+ - 구현되지 않음
  • %% - 구현되지 않음( % 사용)


확장된 파일 이름 형식:


확장된 파일 이름 형식은 토큰을 사용합니다. 토큰과 텍스트 또는 유효한 파일 이름 문자를 활용하여 정확한 파일 이름을 구성할 수 있습니다. 잘못된 문자는 다음과 같습니다. | * : < > ? \ / " 영어(en-US) 토큰은 모든 로캘에서 사용할 수 있습니다. 로캘별 토큰도 사용할 수 있으며 영어 표 아래에 나열됩니다.

예:

${subject}-${date}-${index} - 기본 형식
${prefix}_${date_custom}_${sender} - 기본값 대신 사용자 정의 날짜 형식을 사용합니다.
${subject}-GmailMsg-${date}-${index} - 일부 맞춤 텍스트 포함


토큰 설명
${subject} 메시지의 제목입니다. 길이는 제목 길이 옵션에 따라 제한될 수 있습니다.
${sender} 메시지 보낸 사람의 이름입니다.
${recipient} 메시지 수신자의 이름입니다.
${sender_email} 메시지 보낸 사람의 이메일 주소입니다.
${recipient_email} 메시지 수신자의 이메일 주소입니다.
${smart_name} 메시지가 보낸 편지함 폴더에 있는 경우 받는 사람, 그렇지 않은 경우 보낸 사람 이름
${index} 메시지 인덱스 번호입니다.
${prefix} 접두사 텍스트 옵션 값입니다.
${suffix} 접미사 텍스트 옵션 값입니다.
${date} 표준 날짜 형식입니다. 기본 형식은 YYYYMMDDHHmm입니다.
${date_custom} 사용자 정의 날짜 형식입니다. 사용자 정의 날짜 형식 섹션을 참조하세요.

신규 및 실험적 기본 설정(옵션)

옵션 대화 상자 내에서 아직 구성할 수 없는 몇 가지 새로운 기능과 옵션이 있습니다. 이러한 기능이 안정화되면 UI에 추가됩니다.
이를 사용하려면 구성 편집기 내에서 설정해야 합니다. 편집기에 액세스하려면 도구/옵션 메뉴를 사용한 다음 고급 섹션을 선택하면 하단에 '구성 편집기' 버튼이 있습니다. 검색창에 'importexporttoolsng'를 입력하면 모든 확장 프로그램 기본 설정이 표시됩니다. 그런 다음 필요에 따라 아래의 다양한 기본 설정을 편집할 수 있습니다.

importexporttoolsng.export.index_date_custom_format

이 기본 설정은 인덱스 파일에 사용되는 것과 동일한 전체 형식을 따릅니다. 기본적으로 빈 문자열입니다. 형식이 지정된 날짜/시간 문자열이 있으면 인덱스 날짜 형식이 변경됩니다.

importexporttoolsng.export.attachments.filename_extended_format

importexporttoolsng.export.embedded_attachments.filename_extended_format

이러한 기본 설정은 모두 내보낸 메시지 파일 이름이 사용하는 것과 동일한 유형의 확장 파일 이름 지정 형식을 사용합니다. 첫 번째는 외부 첨부용이고 두 번째는 내장(인라인) 첨부용입니다.

importexporttoolsng.experimental.use_delivery_date

일반적으로 Thunderbird에서 메시지의 '날짜'는 보낸 사람의 메일 클라이언트에서 설정한 날짜입니다. 이 옵션을 'true'로 설정하면 수신자의 메일 클라이언트에서 수신된 날짜를 활용합니다. 이 날짜는 파일 이름과 인덱스 파일의 날짜 모두에 사용됩니다.

기본, 전역 단축키(단축키) 기능

importexporttoolsng.experimental.hot_keys

이 새로운 기본 설정을 통해 사용자는 최대 10개의 키보드 단축키를 설정할 수 있습니다. 바로가기는 대부분의 ImportExportTools NG 기능과 기타 Thunderbird 기능을 수행할 수 있습니다. 기본 설정 문자열은 JSON 형식 개체로 형식화됩니다. 형식과 키는 다음과 같습니다.

단축키 항목 예시(JSON 객체 배열)

id - 키 인덱스(1-10)
key - 키(단일 문자)
keycode - 'VK_*'는 키코드를 정의합니다.
키 또는 키코드 중 하나가 포함될 수 있지만 둘 다 포함될 수는 없습니다.
modifiers - 키 수정자, 공백으로 구분된 목록('control', 'shift', 'alt', 'accel')
oncommand - 직접 명령 문자열
contexts - 키가 활성화된 창 컨텍스트
'all' - 메신저 아래의 모든 창, 작성, 검색
'messenger' - 메인 메신저 창과 그 하위 창
'compose' - 메시지 작성 창
'search' - 검색 대화상자

[{"id": "1", "key": "K", "modifiers": "control shift", "oncommand": "alert('Compose alert')", "contexts": "compose"}, {"id": "2", "keycode: "VK_F1", "modifiers": "control shift", "oncommand": "alert('testD')", "contexts": "all"}]

[{"id": "1", "key": "P", "modifiers": "control shift", "oncommand": "goDoCommand('cmd_printpreview')"}, {"id": "2", "key": "D", "modifiers": "control shift", "oncommand": "exportSelectedMsgs(5)"}]
